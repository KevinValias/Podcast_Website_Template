HTML Structure

Based on HTML5, Soundcast has a simple and compreensive structure, without complications.

Soundcast uses the Bootstrap framework to make things easy. Based on system grid with 12 columns and 1170px wide, everything is mobile friendly. When page width reaches 767px (viewport) the columns become fluid and stack vertically. Visit http://getbootstrap.com/ to check more about Bootstrap.

All HTML pages have been validated on W3 Validator.

Header Static, Absolute and Sticky

In this theme we have 3 types of header, but, the only difference between the types is the position.

Static

The headers occupy a space in the page and don't move with scroll. To check an example, access the static-header.html in Theme.
To make your header static, just add the "static" class (remove "absolute" or "sitcky" if need).

Absolute

Found on majority pages (check index.html), the absolute header doesn't occupy space and is placed in top of the page, but don't move with mouse scroll.
To make your header absolute, just add the "absolute" class (remove "static" or "sticky" if need).

Sticky

Sticky doesn't occupy space in page, and, when scroll, the header follows the page scroll on the top. You can check this at sticky-header.html. In sticky header, we have a "data-offset" attribute to set an height value. This height is the amount of scroll needed to add a opacity background to header.
To make your header sticky, just add the "sticky" class (remove "absolute" or "static" if need) and the "data-offset".

Podcast Cards

The podcast cards is how we list the podcasts. For more customization, we have 3 types: Full, Boxed and Sidebar.
The HTML structure of the types is the same, the only difference is the control class "full", "boxed", "sidebar".

Full

If you need a fluid content, the full type is the best option. Just pick the "podcast-card" HTML structure and add the "full" class.

Boxed

With 2 columns, the boxed type is the "modern" way option. Just pick the "podcast-card" HTML structure and add the "boxed" class.

Sidebar

If you need to list a podcast in a sidebar, just use the "sidebar" type. Pick the "podcast-card" HTML structure and add the "sidebar" class.

Podcast Player, SoundCloud and Video

Thinking in customization, with Soundcast you can choose between three categories: Custom Player, SoundCloud and Video.

The Custom Player is a JavaScript library called mediaelement.js and theres a lot of documentation around this. The essential is already done, but if you need some customization, check: http://mediaelementjs.com/.

For SoundCloud and Video (Vimeo, Youtube), these platforms uses the iframe for better integration. If you want to use this category, just get the embed code from the platforms :)

MailChimp Newsletter

The MailChimp integration is a simple form action that send data to the real MailChimp form. Follow the steps to get your form action:

  1. Log in your MailChimp account
  2. Click in "Lists" tab
  3. Choose one of your Lists (or create one), open the options and choose Signup Forms
  4. Select "Embedded forms"
  5. Click on the "Horizontal" tab
  6. In the "Horizontal" tab, scroll down and copy the form action on the code generated
  7. Now, just paste this action on the Soundcast newsletter form and is done!

Image Gallery (Single Page, Podcast)

The structure to image gallery is simple, don't have secrets. The library behind this is magnific-popup.js.

<a href="assets/img/gallery/large/image.jpg" class="gallery-zoom"> <img src="assets/img/gallery/thumb/image.jpg" alt="Gallery" title="Gallery" /> </a>

Note: Don't forget the "gallery-zoom" class, it does the trick.

CSS

For better organization, we use LESS to made things easier.

How to use LESS

Less is a CSS pre-processor, meaning that it extends the CSS language, adding features that allow variables, mixins, functions and many other techniques that allow you to make CSS that is more maintainable, themable and extendable.
LESSCSS.org

Code CSS with LESS is so simple and powerfull. Doesn't know how LESS works? The official website shows how to: LESS Feature

LESS Modules Structure

The LESS folder is separated by modules with semantic names

- LESS
  -- style.less (Import all other LESS files)
  -- reset.less
  -- assets (mostly, elements found across the theme)
  -- helpers (mixins, colors and responsive variables)
  -- structure (margins and clearfix)
  -- template (general theme css, like footer, header, all)
  -- texts (titles and texts)

Theme CSS (Without LESS)

For those who don't want to use LESS, the theme CSS is already generated in "assets/css/style.css". Just use like always :D
The CSS is well coded and commented, with good class semantics.

Fonts

All the texts fonts present in this theme is from Google Fonts. In Soundcast we use Open Sans and Montserrat. Feel free to change.

JavaScript

Soundcast theme uses jQuery as your main library. All JavaScript files are included in JS folder.
The Soundcast JavaScript file is called main.js and has all other plugins calls and instances.

If you want to edit some variables, feel free to open the main.js file, but take care, some plugins has her own methods and names.

PHP

In Soundcast, the contact form is almost done, we just need you to configure one variable. To do that, follow the steps below:

Open the PHP file contact.php (php/ folder) and go to the line 19. Here, just alter the $address value to your contact email. Yes, just that.

After that, your contact form is ready!
